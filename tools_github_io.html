Хостинг на github.io

GitHub надає безкоштовний хостинг для ваших проектів (GitHub Pages). На кожен ваш логін можна створити один сайт виду:

https://ваш_логін.github.io

Цей сайт ви можете використовувати собі як портфоліо, викладати туди свої проекти, свою інформацію.

Свої лекції я також тримаю на цьому сервісі.

Даний хостинг не має бекенду - ви не можете використовувати базу даних, серверні скрипти та служби, зборщики і т.п. Лише фронтенд-частину: окремі веб-сторінки, стилі, зображення, документи, js-скрипти та JSON-файли.

Отже, приступимо. Зайдіть на github.com, створіть новий проект (репозиторій), клацнувши по плюсику вгорі справа і задайте йому ім'я такого виду:

ваш_логін.github.io

// наприклад:
poshtarboba.github.io

Перше слово - ваш логін (його ви можете побачити правіше - Owner). Далі йде крапка і домен github.io.

Ви не можете створювати довільні піддомени на GitHub Pages, лише один піддомен на один аккаунт.

Назва репозиторія hello.github.io не призведе до створення відповідного сайту, якщо ваш логін не hello.

В папці work створіть папку з назвою вашого репозиторія: ваш_логін.github.io. Цю папку не треба ховати в підпапках років і місяців, скоріш за все ви будете часто працювати з цим репозиторієм, хай він буде завжди під рукою.

Клонуйте репозиторій в папку work (підпапка з назвою репозиторія створиться сама):

d:\work> git clone https://github.com/ваш_логін/ваш_логін.github.io.git

Створіть index.html (не забудьте про кодування UTF-8, якщо користуєтесь Notepad++):

<meta charset="utf-8">
<h1>Привіт. Це сайт і портфоліо Васі Пупкіна</h1>

Заляйте файл на GitHub:

git add .
git commit -m "hello github.io"
git push

Зачекайте секунд 30-60 і спробуйте зайти на ваш новий сайт https://ваш_логін.github.io

Окремі репозиторії як папки проекта github.io

Наприклад, ви робите портфоліо на гітхабі і хочете викласти свої роботи в папки:

vasyapupkin.github.io:
project1/
project2/
magazin-obuvi/
index.html
style.css

В цьому випадку ваш основний репозиторій швидко розростеться до великих об'ємів, його адміністрування буде неефективним. Окрім того, GitHub має обмеження на розмір репозиторія (станом на 2018 рік - 1 Gb), але кількість репозиторіїв необмежена.

Основний репозиторій тримайте в чистоті, без зайвих файлів:

vasyapupkin.github.io:
index.html
styles.html

Кожну роботу заливайте в окремий репозиторій. Створюйте нову гілку від master і заливайте на гітхаб:

git checkout -b gh-pages
git push origin gh-pages

Відкрийте налаштування репозиторія на GitHub (вкладка Settings) і переконайтесь, що поточний репозиторій став підрозділом вашого проекту github.io:

Порада: зручно називати репозиторії проектів для вашого портфоліо наступним чином:

рік-назва_проекту
рік-місяць-назва_проекту
2017-project1
2017-project2
2018-04-magazin-obuvi
2018-04-beer_world
2018-07-multiplex

Практика

  1. Створіть на GitHub репозиторій test-io
  2. Клонуйте його в папку поточного місяця в своїй робочій папці.
  3. Створіть файл index.html: <meta charset="utf-8">
    <h1>Це підпапка test-io мого проекта на github.io</h1>
  4. Закиньте файл на сервер, створіть гілку gh-pages і її також запуште: git add .
    git commit -m "make subpage for my github.io"
    git push
    git checkout -b gh-pages
    git push origin gh-pages
  5. Перевірте налаштування репозиторія на GitHub.
  6. Зайдіть на новостворену сторінку https://ваш_логін.github.io/test-io/
